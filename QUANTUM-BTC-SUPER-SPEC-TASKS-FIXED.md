```markdown

  - [ ] 1.2 Create `btc_hourly_snapshots` table
```

### Key Formatting Rules Applied

1. **Top-level tasks** must use format: `- [ ] N. Task Name`
2. **Sub-tasks** must be indented with format: `  - [ ] N.M Sub-task Name`
3. **Optional tasks** (tests) marked with `*`: `  - [ ]* N.M Optional Task`
4. **Maximum 2 levels** of hierarchy (parent + children)
5. **No standalone headers** like `### Task N:`
